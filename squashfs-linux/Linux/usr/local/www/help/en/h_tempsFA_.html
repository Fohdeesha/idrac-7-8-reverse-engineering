Temperatures

Use this page to:

  • View the status and readings of the on-board ambient temperature probes. All readings are displayed in Centigrade (°C) and Fahrenheit (°F) units. The temperature probes may be system dependent. The minimum and maximum temperature thresholds for warning and critical states are displayed, along with the current health status of the probe.


  • View if the system is Fresh Air compliant or not and monitor the percentage of time the system has spent operating in an environment, where the ambient temperature is greater than the normally supported temperature threshold. The system board inlet temperature sensor reading is collected over a period of time to monitor the temperature. See System Board Inlet Temperature Historical Data.
    • For more information on Fresh Air cooling, see the iDRAC7 User's Guide.

System Board Inlet Temperature Historical Data

This section provides a graphical display of the stored inlet temperature (average and peak values) for the last day, last 30 days, and last year. By default, the data for the last year is displayed. The data collection starts when the system is first powered on after it is shipped from the factory. The data is collected and displayed for the duration when the system is powered on. For more information on Fresh Air cooling, see iDRAC7 User's Guide.

The graphs are populated with:

  • 24 data points for the last 24 hours (average and peak values for each hour)
  • 30 data points for the last 30 days (average and peak values for each day)
  • 52 data points for the last 52 weeks (average and peak values for each week)

When you hover over each point, the average and peak values with the number of hours, days, or weeks that has elapsed from the current hour, day, or week respectively is displayed within the graph.

The graph drops to the OFF position when the system is powered off.

Note:The historical data may be available even when the system is powered off, only the last few hours may not be available.

You can export the stored inlet temperature data of the last seven years to an XML or CSV file. The data is collected twice per day. After selecting the file type, click Export. The Save As dialog box is displayed. Specify the location to save the file and click OK. For each recorded inlet temperature data, the timestamp, average and peak values are available. At any time, if the system was powered off, then the reading is -128 degree centigrade (that is, data is unavailable). At a time, only one user can export the data.

Total Operation Time Indicates the number of days the system has been in operation. The maximum value is 365 days.
Time Spent in Warning Threshold Range Displays the time the system has spent in the warning threshold range:
  • Green — No time is spent in range 0%.
  • Amber — Time spent is in the range of 1-7%.
  • Red — Time spent is more than 8%.
Time Spent in Critical Threshold Range Displays the time the system has spent in the critical threshold range:
  • Green — No time is spent in range 0%.
  • Amber — Time spent is in the range of 0.1-0.7%.
  • Red — Time spent is more than 0.8%.
Note:If the system has spent more than 10% of the time in the warning threshold range or more than 1% of the time in the critical threshold range, events are logged in the Lifecycle Log and SNMP traps are generated. You can also set the alert recurrence. To do this, go to Overview→ Server→ Alerts→ Alert Recurrence.
