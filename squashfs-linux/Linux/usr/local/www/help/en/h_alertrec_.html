Alert Recurrence
Jump to: Other Options

Use this page to generate an event if the system continues to operate above the normally supported temperature threshold for a specified operational time. If the average temperature over the specified operational time, is greater than the warning level (> 10%) or the critical level (>1%), then an event is logged in the Lifecycle Log and the corresponding SNMP trap is generated. You can view the historical data of the inlet temperature in the Temperatures page, go to Overview→ Server→ Power/Thermal→ Temperatures.

Category Displays the alert category as System Health.
Alert Displays the alert type as Temperature Statistics.
Severity Indicates the severity—Warning or Critical. For more information, see iDRAC7 User's Guide.
Recurrence Enter the alert frequency value in days. Valid values: 0-365. Default is 30. A value of 0 indicates the event recurrence is disabled.
Note:You must have administrative privileges to set this value.
